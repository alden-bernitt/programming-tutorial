Data and Variables

Background

While all programs can be viewed as a sequence of statements, there is another key element involved: data. Data are the values that statements operate upon.

Examples

  • Consider the following addition operation: 4 + 5
    The "statement" is addition, and the "data" are the values 4 and 5.

  • In the "Hello, world!" program from the Programming Basics guide, we used the single print statement print("Hello, world!"). The data we used with this statement was the text "Hello, world!".

Data Types

In accordance to how we view data in the real world, programming languages use data types to distinguish between the various types of data. Each data type is defined by a set of possible values and a set of operations to perform on those values.

Examples

  • When we count objects (such as sheep), we do so using a set of numbers known as integers. Integers include values such as 1, 9001, 0, and -69. Operations on integers include addition +, subtraction -, and multiplication *
    Notably, integer values do not include values such as 0.5 and the square root of 2 (often denoted sqrt(2)). Operations that are not defined on integers include the usual version of division ÷ (this is because, for example, the result of dividing the integers 1 and 2, that is 1 ÷ 2, results in a value that is not an integer).

Python provides a rich set of data types to make the job of the programmer easier. As we introduce the following data types, note that we do not describe all of the data types provided by Python, nor do we describe all operations included with each data type. However, take heart in that many of these data types will be introduced in later guides, and we will give you the tools necessary to perform your own research when the time comes.

As you read the following sections, it may be helpful to try your own examples using a Python interpreter.

Integer

The integer, or int, data type represents the set of integers, and includes values such as 1, 9001, 0, and -69. In mathematics, this set might be denoted {..., -1, 0, 1, ...}. Python provides the expected operations on integers, along with a few obscure ones.

Integer Operators in Python
Op Description Example
+ Addition 9 + 5 = 14
- Subtraction 9 - 5 = 4
* Multiplication 9 * 5 = 45
// Floor Division 9 // 5 = 1
** Exponentiation 9 ** 2 = 81
% Remainder 9 % 5 = 4

The most noteworthy of these are the // and % operators. The // operator performs an operation known as floor division. The result of this operation is equal to the division of the first operand by the second rounded down. The % operator, called the modulo operator, essentially produces the remainder of the first operand divided by the second operand.

You might find that the % operator produces strange results when one or both of the operands are negative. This is because the operation a % b actually performs the calculation a - b * floor(a / b) When a and b are both positive, this is identical to calculating the remainder of a ÷ b.

We recommend testing // and % using a positive and negative value, a negative and positive value, and two negative values. You may be surprised at the results. See if you can figure out why the results are what they are.

Float

The float data type represents the set of real numbers. This set includes integer values, such as the ones shown previously, as well as decimal values, such as 1.5, -99.99, and 3.14159. The float data type supports mostly the same operations as the int data type. Notably, float values also support division.

Float Operators in Python
Op Description Example
+ Addition 2.75 + 1.25 = 4.0
- Subtraction 2.75 - 1.25 = 1.5
* Multiplication 2.75 * 1.25 = 3.4375
// Floor Division 2.75 // 1.25 = 2.0
/ Division 2.75 / 1.25 = 2.2
** Exponentiation 2.75 ** 1.25 = 3.5413...
% Remainder 2.75 % 1.25 = 0.25

String

The final data type we will discuss in this guide is the string data type. A string is simply a sequence of characters, and represents text. Strings in Python are denoted by characters enclosed in either single quotes ('), or double quotes ("). Examples include "Beans", 'To be, or not to be', and "" (this is a sequence of zero characters, and is referred to as an empty string or null string).

What happens if we want to place quotes within a string? How will the interpreter know which quotes are supposed to close a string and which quotes are supposed to be part of the string? Consider the example "I had a "great" time". The Python interpreter would see two strings in this case: "I had a ", and " time". These questions are answered in the next section.

String Operators in Python
Op Description Example
+ Concatenation 'Py' + 'thon' = 'Python'
* Repetition 'Py' * 3 = 'PyPyPy'

The concatenation operation combines its two string operands into one string by appending the second string directly to the end of the first. The repeat operation takes one string operand and one integer operand, and produces a new string by repeating the original string the specified number of times.

What if we want a string to contain special characters such as a tab character or line break character? What if we want to include quotes within the string itself? To accomplish this, we use something called an escape sequence. In Python, along with many other programming languages, an escape sequence begins with a \ character followed by one or more other characters which specify the special character.

Common Escape Sequences
Sequence Description
\n Newline (line break)
\t Tab
\" Double quote
\' Single quote
\\ Backslash

Examples

  • To embed a quote within a string, we could write "Dwayne \"The Rock\" Johnson". We might use this string with a print() statement to print a string containing quotes. 

    >>>print("Dwayne \"The Rock\" Johnson")
Dwayne "The Rock" Johnson

    It's worth mentioning, however, that the same result can be achieved by using single quotes to quote the string instead of double quotes. By doing so, it is no longer ambiguous which quote closes the string. 

    >>>print('Dwayne "The Rock" Johnson')

Variables

Variables are one of the most fundamental concepts in computer programming. Conceptually, In Python, a variable is a name that we give to a storage unit which stores a single value of a single data type. That is, we initialize a name with a storage unit containing a value. You might visualize this as asking Python to give you a box and placing the value inside, and sticking a label on it with your chosen name. In Python, this entire process is done using the assignment (=) operator. Once we have created a variable, we are able to assign any new value it, and we are able to use it in operations as if it were a plain value.

Examples

  • >>>name = "John Doe"
>>>age = 35

    Note that the first statement assigns a name to a string, and the second statement assigns a name a to an integer.

    These statements may be visualized as follows.

  • We are allowed to assign new values to variables, even if the data types are different. 

    >>>x = "John Doe"
>>>x = 35

  • We can use variables as part of operations, as if they were regular values. 

    >>>width = 3
>>>height = 4
>>>area = width * height

    Once the third statement executes, we will have a variable named area containing the value 12.

Although in mathematics = is used to mean equality, many programming languages do not share these semantics. This is the reason why statements such as x = x + 1 make sense in Python and other programming languages. In words, this statement means "retrieve the value stored in x and add 1 to it, then assign the name x to that new value.

If x originally contained the value 4 before the statement executed, it will contain the value 5 after.

It is sometimes useful to distinguish between variables and literals. Simply put, literals are plain values such as 7 (an integer literal), or 3.14 (a float literal), and "Python" (a string literal). Variables, on the other hand, refer to the names storing those values, such as x, num, or area.

Variable Names

There are rules as to what constitutes a valid identifier (i.e. name). Although the rules are rather complicated, let it suffice to say that identifiers may only contain letters, digits, and underscores (_), and must begin with either a letter or underscore. Furthermore, reserved words may not be used as identifiers. We will learn many reserved words in time, but if you are curious now, you can reference the official Python website for a list.

Although not strictly necessary, we highly recommend that you adhere to the following naming conventions.

    Do

  • Use only English letters (capital and lowercase A through Z), numbers, and underscores.

  • Begin an identifier with a lowercase letter.

    Do Not

  • Begin an identifier with an underscore (e.g. _num)

  • Use foreign characters (e.g. すうじ)

  • Use predefined names (e.g. print)

  • Begin an identifier with a capital letter (e.g. Num)

As always, some exceptions will be made in some circumstances, but that will have to wait until you have sufficient knowledge and experience.

Expressions

Expressions are a group of operations that evaluate to a single value. We can assign the results of expressions to variables, or use them in statements like the print() statement. Expressions can contain variables, literals, or a mixture of both.

Just like mathematics (recall PEMDAS ), Python specifies a precedence for its operators. Operators with higher precedence are evaluated before operators with lower precedence. Like in mathematics, operations that are enclosed in parentheses are treated with the highest precedence. A table displaying the precedence of some of Python's operators is shown below.

Operator Precedence (Highest to Lowest)
Operator Description
() Parentheses
** Exponentiation
*, /, //, % Multiplication (repeat), division, floor division, remainder
+, - Addition (concatenation), subtraction

Operators with the same precedence are evaluated from left to right with the exception of the exponentiation operator, which is evaluated from right to left.

Examples

  • >>>2 * 3 + 4
10

  • >>>2 * (3 + 4)
14

  • >>>2 ** 3 ** 2
512

  • >>>(2 ** 3) ** 2
64

  • >>>3 * 'Py' + 'thon'
'PyPyPython'

  • >>>3 * ('Py' + 'thon')
'PythonPythonPython'

  • >>>x = 3
>>>y = x * 2
>>>x + y
9

  • >>>fruit = 'apples'
>>>print('I really love ' + fruit)
I really love apples

Input

So far, every program we've written executes exactly the same every time it's run, which is boring. Now that we know about variables, we can take input from the user and store it in a variable to alter the results of a program. In Python, we can read (i.e. receive) user input by using the input() function, and store it in a variable as is done with the statement name = input(). When this statement is executed, the program will wait until text is available on standard input, and then store that text in name. If we are running this program at a terminal, we can make text available to standard input by typing into the terminal and pressing the Return (i.e. Enter) key.

Choosing the variable identifier name suggests that the programmer is expecting the user to supply the program with some kind of name. The user, however, has no way of knowing this. To solve this problem, we can supply a prompt to the user to guide them when entering a value. This is done by including a string value in between the parentheses of input(). For example, it may be helpful to use input("Please enter your first name: "). This prints the given text to standard output before waiting for the user's input. By including this prompt, the programmer can communicate to the user the kind of input that is expected from them.
